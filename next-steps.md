## חלק ב': צעדים מיידיים (היום)

### צעד 1: אמת שה-RLS תוקן
הרץ ב-Supabase SQL Editor:
```sql
-- בדוק אם הפוליסיות קיימות
SELECT policyname FROM pg_policies WHERE tablename = 'tasks';
```
אם אין `dev_tasks` - הרץ את המיגרציה מ-`002_fix_team_members_rls.sql`

### צעד 2: בדוק פונקציונליות
1. `cd flowplan && npm run dev`
2. פתח http://localhost:3000
3. צור פרויקט → צור שלב → צור משימה
4. אם יש שגיאה - שלח לי את הטקסט מ-DevTools (F12)

## חלק ד': איך לעבוד עם Claude Code יותר טוב 🚀

### 1. תן לי הקשר מלא
```
❌ גרוע: "תתקן את הבאג"
✅ טוב: "כשאני לוחץ על 'צור משימה' מופיעה השגיאה הזו: [הדבק שגיאה]"
```

### 2. שלח צילומי מסך ושגיאות
- **F12 → Console** - העתק שגיאות אדומות
- **צילום מסך** - אני יכול לראות תמונות!
- **טקסט מלא** - לא לקצר הודעות שגיאה

### 3. השתמש בפקודות מובנות
```bash
/plan          # תכנון לפני שכותבים קוד
/commit        # ליצור commit מסודר
/review        # לבקשת code review
/init          # ליצור CLAUDE.md חדש
```

### 4. בקש ממני להשתמש ב-Agents
```
"תשתמש ב-planner agent כדי לתכנן את הפיצ'ר"
"תעשה code review עם ה-agent"
"תריץ את הטסטים עם tdd-guide"
```

### 5. תן לי לחקור לפני שאני כותב
```
✅ "תחקור איך עובד ה-authentication ב-codebase לפני שנתחיל"
✅ "תבדוק אילו קבצים צריך לשנות בשביל להוסיף את הפיצ'ר"
```

### 6. עבוד בצעדים קטנים
```

### 7. תגיד לי מתי משהו לא עובד
```
"הרצתי את מה שנתת לי וזה לא עבד - הנה השגיאה: [...]"
